# Documentación — La Termal

Este directorio concentra la documentación operativa y técnica del sistema.

## Lectura recomendada (orden)
1) **ESTADO_ENTREGA.md** — qué está listo hoy, qué falta, riesgos.
2) **CHECKLIST_PRODUCCION_LOCAL.md** — checklist para instalar y operar en entorno local.
3) **INSTALACION_SERVIDOR.md** — guía de instalación (Windows/Linux).
4) **OPS_WINDOWS_SERVIDOR.md** — operación en Windows (tareas, firewall, backups).
5) **REPORTES_EMAIL.md / REPORTES_EMAIL_SETUP.md** — reportes gerenciales y envío por correo.
6) **AUDITORIA.md** — auditoría (filtros, export CSV, reportes).
7) **ACCESOS_Y_ROLES.md** — roles, permisos y visibilidad por usuario.
8) **PLAN_PRUEBAS_MANUALES.md** — plan de pruebas manuales (evidencia de entrega).
9) **HISTORIAS_USUARIO.md** — backlog y criterios de aceptación.

- **FLOTA_CHOFERES.md**

## Notas importantes
- Los directorios `backup_patch_*` y carpetas temporales de parches **no deben quedar en el proyecto**.
- La carpeta `reports/` genera archivos **runtime** (no se versionan).
